Jonathan Walberg, University of Virginia

## Replication Steps

1. **Collect prediction market data:**
   ```bash
   python3 collect_polymarket.py   # ~5 min; fetches from Gamma + CLOB APIs
   python3 collect_kalshi.py       # ~3 min; fetches from Kalshi REST API v2
   ```

2. **Run event study analysis:**
   ```bash
   python3 event_study.py
   ```
   This generates all tables and figures in `results/`.

3. **Compile paper:**
   ```bash
   pdflatex BettingOnWar.tex
   bibtex BettingOnWar
   pdflatex BettingOnWar.tex
   pdflatex BettingOnWar.tex
   ```

## Data Sources

- **Polymarket**: Gamma API (market discovery) + CLOB API (price history)
  - API docs: https://docs.polymarket.com
  - No authentication required for public endpoints
  - Markets span 2022-12 to 2024-01, covering key geopolitical events
- **Kalshi**: REST API v2 (market listing + candlestick history)
  - API docs: https://docs.kalshi.com
  - No authentication required for public market data
  - Note: Most geopolitical markets may require specific series tickers
- **Event timeline**: 17 crisis events hand-coded from news sources
  - Embedded in `event_study.py` (GEOPOLITICAL_EVENTS list)
  - Covers: Russia-Ukraine war, Iran-Israel escalation, US strikes, etc.

## Notes

- If API access is restricted, `event_study.py` automatically generates synthetic data for demonstration.
- The synthetic fallback preserves analysis structure but should not be used for substantive conclusions.
- Polymarket CLOB data is only available for markets created after the CLOB migration (~2022).
- Kalshi's public API is dominated by sports markets; geopolitical markets require known series tickers.
- All random seeds are fixed for reproducibility (numpy seed 42 for synthetic, 123 for placebo).
